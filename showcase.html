the core idea

Every argument has two layers.

The surface layer is the claims, evidence, and studies people actually trade. The prior layer is the implicit probability assignments and value weights that decide, in advance, which claims feel obvious and which feel absurd.

same priors

When two people share the same priors, the same evidence pushes them to the same conclusion. Trading more studies works — it's actually doing something.

different priors

When priors diverge, evidence gets filtered through them on the way in. Both sides can read the same paper and walk away more certain of where they already stood. No amount of evidence-trading closes that gap, because the gap was never about the evidence.

why this and not a summarizer

The moat isn't the extraction. It's what you can do with the output.

Any model can summarize two arguments. The useful part is turning the summary into a structured object you can actually reason about.

Quantified, not qualitative

Every shared claim gets a confidence number from each side. A 65-point gap isn't a vibe — it's 15% against 80%, and you can see exactly which disagreement is the largest one in the room.

Empirical vs. normative, as a real output

The single most useful fact about any disagreement: can evidence close it, or not. Most debate tools never make this distinction. It's arguably the only one that matters.

Built for the messy version

Voice-first, real-time capture means it works on an actual conversation in progress, not a polished essay written after the fact. Messier arguments are more honest ones.

A map, not a verdict

It tells each side what they'd need to update and by how much. It does not tell anyone who's right. That's a deliberately different — and more durable — kind of output.

under the hood

The reasoning was always right. The rendering wasn't, until now.

Small local models reliably get the content right and occasionally drift on exact casing — "Surface" instead of "surface". The old pipeline matched on that string exactly, so one wrong letter dropped an entire layer of real reasoning into the void.

before

The model returned the full reasoning. A casing mismatch on the level field meant three of four rows matched nothing and rendered nothing — silently.

after

The Ollama path now uses grammar-constrained decoding, so the model is structurally unable to emit anything outside the enum. The Claude path forces tool-use against the same schema instead of parsing free text.

A client-side normalization pass sits underneath both as a second line of defense, and if a model genuinely returns nothing for a layer, the interface says so directly — "the model didn't return a premise breakdown — try re-running" — instead of three quiet rows of nothing.
